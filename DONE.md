Original problem: prior attempts (flickering box-shadow glow; box-shadow ruffled by an SVG turbulence/displacement filter; teardrop outlines rippled by `#flame-distort` + opacity flicker) all looked bad. A first rework (outward-pointing tongues rising/swaying via `scale`/`rotate`, tip-fade mask, `drop-shadow` halo) was also rejected — the side/bottom tongues pointing sideways and downward didn't read as fire. FINAL design (this entry): all flames point UP and completely surround the card, with a glow around the whole edge, and is built for performance first. As built: `flameTier(streak)` returns the glow `rgb` triple plus a `gradId` selecting one of three pre-defined vertical SVG gradients (`FLAME_GRADIENTS`, bright hot base → tier color → transparent tip) rendered once into a hidden `<defs>` — so a single flat-filled path per tongue reads as a glowing flame with no per-element gradient. `FLAME_TONGUES` rings the card with 19 upward-pointing tongues (single `FLAME_PATH`, 30×70 viewBox, wide rooted base → wavy point): a tall crown across the top, climbers up both sides, and risers from beneath the bottom (anchored at `top` > 100%). The flame layer sits behind the card (`z-0`, Image `relative z-10`) so each tongue's inner half tucks out of sight. Performance: the ONLY animated properties are `transform`/`opacity` (compositor-only) — `flame-lick` rises + flickers each tongue (staggered via per-tongue `animation-delay`/`animation-duration`) and `flame-glow-pulse` breathes the `.flame-glow` box-shadow halo's opacity; the soft flame edge is a STATIC `filter: blur(1.4px)` rasterized once (not re-run by the transform), and `will-change` promotes each layer. No filter or mask runs per frame. Removed the inline `<filter id="flame-distort">` SVG and the old turbulence/mask/drop-shadow CSS. Streak-tracking logic (`streak`/`streakCardId`) and per-tier colors unchanged. Files: `src/app/compare/ComparisonScreen.tsx`, `src/app/globals.css`.
